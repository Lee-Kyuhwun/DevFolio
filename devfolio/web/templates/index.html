Advanced Import

YAML/JSON은 일반 입력 방식이 아니라 기존 데이터 이관용입니다. 필요하면 devfolio data import examples/connected_car_gateway.yaml 를 사용하세요.

Serve-First Workflow

붙여넣기에서 문서 생성까지 한 흐름으로

CLI CRUD는 그대로 남겨두고, 일반 사용자 경로는 이 화면 안에서 끝나도록 정리했습니다.

Git 저장소 자동 분석

Git Scan

로컬 git 저장소 경로를 입력하면 본인 커밋을 분석해 포트폴리오 프로젝트를 자동으로 생성합니다.

Docker 사용 시: 홈 디렉터리가 컨테이너 내 /home/user로 자동 마운트됩니다.
예: 로컬 경로가 ~/projects/my-app이면 → /home/user/projects/my-app 입력
다른 경로를 사용하려면 .env에 REPOS_DIR=/원하는/경로를 추가하세요.

스캔 결과

Step 1

작업물 붙여넣기

프로젝트 설명, 메모, 회고, PR 정리, 노션 내용 등 편한 형태로 붙여넣으세요.

저장 포맷은 내부적으로 YAML이지만, 이 화면에서는 YAML을 직접 작성할 필요가 없습니다.

Step 2-3

초안 검토 및 편집

AI가 만든 초안은 바로 저장되지 않습니다. 수정한 뒤 저장 또는 미리보기를 선택하세요.

Library

저장된 프로젝트

저장된 프로젝트를 불러와 다시 편집하거나, 저장된 상태에서 AI 문구를 갱신할 수 있습니다.

Selected

다음 단계

프로젝트를 에디터로 불러오거나, 저장된 데이터 기준으로 바로 요약과 bullet을 재생성할 수 있습니다.

저장된 프로젝트가 아직 없습니다. Intake 탭에서 초안을 저장하세요.

Step 4

Preview & Export

저장 전 draft 또는 저장된 프로젝트 기준으로 resume/portfolio를 확인하고 내보낼 수 있습니다.

Selection

저장된 프로젝트 선택

Saved source를 선택한 경우 어떤 프로젝트를 문서에 포함할지 지정합니다.

아직 생성된 미리보기가 없습니다.

Rendered

문서 미리보기

이 화면은 저장 전 draft도 그대로 렌더링합니다.

Preview 버튼을 눌러 문서를 생성하세요.

Settings

사용자 프로필

문서 상단에 노출될 기본 정보입니다.

AI

AI Provider

현재 프로젝트는 설정 후에만 AI intake와 문구 생성을 실행합니다. 수동 편집만으로도 저장과 preview는 가능합니다.

Output

Export / Sync / General

문서 기본 포맷과 GitHub 백업 설정을 한 곳에서 관리합니다.
